## 用户心智模型

把 Know-share 想成 **"公共图书馆目录 + 撮合 + 私下握手"**：用户浏览的是**目录卡片（清单）而非书本身（原始知识）**；凭卡片上的脱敏摘要、来源统计和信任信号判断价值；发起一次交换请求；对方同意后**才互换联系方式、私下握手**完成交付。信任信号一路引导，隐私边界始终可见（追溯 `README.md` 核心理念、`ENT-003/004/011`、`INV-01/03`）。

## 导航模型

- **公共主导航（无需登录）**：发现(`IA-002`)、交换记录(`IA-005`)、信任网络(`IA-007`)、Agent 技能(`IA-008`)、仓库/帮助(`IA-013`)；右侧常驻全局搜索 + 提交模块 CTA(`IA-004`) + GitHub 登录状态(`IA-001`)。
- **已登录增项**：个人中心(`IA-009`)、通知(`IA-010`)、设置(`IA-014`)；具管理员角色时显示审核控制台(`IA-011`)。
- **二级进入**：模块详情(`IA-003`) 由发现/搜索/档案进入；交换详情(`IA-006`) 由交换记录/个人中心进入；用户档案(`IA-007`) 由模块卡片/交换记录进入。
- **全局搜索**：在相应规格就绪后路由到模块、主题、用户或交换（`FR-001`）。

## 内容层级

- **首屏 = 发现(`IA-002`)**：作为公共主入口与价值橱窗（含平台统计）。
- **决策面 = 模块详情(`IA-003`)**：用户在此判断"是否请求交换"，是信息密度最高的公共面。
- **流程面 = 提交向导(`IA-004`)**：线性多步，隐私门为强约束节点。
- **声誉面 = 信任档案(`IA-007`)**：聚合可解释的信任信号。
- **私域面 = 个人中心(`IA-009`)/设置(`IA-014`)**：登录后管理自己的模块、交换、联系方式。

## 进入与退出点

- **进入**：公共发现页（人或 agent）、`IA-012` API（agent 集成）、指向模块/档案/交换的深链接。
- **退出到外部**：交换被接受后转向私有通道（默认 GitHub 私有仓库，`ASM-007`）完成交付；GitHub 登录；外部文档/仓库链接。
- **关键边界**：退出到私有通道前，平台只交换"联系方式 + 状态引用"，不承载内容（`INV-01/04`）。

## 空状态

- 发现页无结果/空注册表：引导提交首个模块或调整筛选（`IA-002`）。
- 无已发布模块：引导走提交向导（`IA-009` → `IA-004`）。
- 无交换记录：解释交换如何发起（`IA-005`）。
- 无通知：说明将在交换/评审/反馈事件时通知（`IA-010`）。
- 无反馈/无信任历史：说明信任如何随交换积累（`IA-007`）。

## 权限敏感界面

| 界面 | 访问要求 | 依据 |
| --- | --- | --- |
| 提交向导 `IA-004` | 需 GitHub 登录 + 各步同意门 | `NFR-005`、`FLOW-007` |
| 发起交换 / 交换详情 `IA-006` | 需登录；联系方式仅在 `Accepted` 后对该次对方可见 | `INV-03`、`FR-130` |
| 设置·联系方式 `IA-014` | 仅本人；默认私密 | `INV-03`、`DEC-010` |
| 审核控制台 `IA-011` | 仅管理员角色 | `FR-100` |
| 私有交付 | 平台外、双方同意后 | `INV-01`、`ASM-007` |
| 公开但匿名可看 | 发现、模块详情(公开部分)、信任档案、Agent 技能、交换记录(脱敏) | `FR-001`、`INV-04` |
