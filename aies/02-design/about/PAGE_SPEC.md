# 页面规格 · about（仓库 / 帮助 / 关于，含平台统计）

## 摘要

本产物把信息架构界面 `IA-013`（仓库 / 帮助 / 关于）落成可构建的页面契约：开源可审计入口、隐私与信任说明、平台聚合使用统计展示（`ENT-019`，仅聚合、不含 PII）、帮助 / FAQ，以及指向可审计规则 / 隐私模型 / 数据契约的链接。视觉真源为 `docs/design/generated/IA-013-about-repo.html`（+ `.png`）。本页是**公开、匿名可看**界面（`IA_SPEC.md` 权限敏感界面表），不要求登录。

## PAGE-100 · 关于 / 仓库 / 帮助整页

### 页面用途

- 作为 `IA-013` 的整页编排契约：在一个公开匿名可看的 `/about` 路由上，自上而下编排五个区块（Hero `PAGE-101` → 平台统计 `PAGE-102` → 隐私与信任 `PAGE-103` → FAQ `PAGE-104` → 可审计链接 `PAGE-105`），并承载全局 AppShell（`FR-001`）与页脚。
- 价值：把"开源可审计"（`NFR-004`）与"平台聚合使用统计"（`FR-140`）在同一个对外门面里讲清，让人或 agent 在不登录、不接触任何私有内容的前提下理解平台的理念、健康度与可审计性。
- 区块拆分理由：该单页信息密度高且区块职责彼此独立（统计来自轻后端 `ENT-019`、FAQ 为静态内容、链接为外部 / 文档跳转），拆成 `PAGE-101`~`PAGE-105` 便于组件规格与测试按区块扇出；`PAGE-100` 只保留"编排 + 整页状态"。

### 上游需求链接

- `IA-013`（仓库 / 帮助 / 关于；主要实体 `ENT-019`）。
- `NFR-004`（开源可审计）、`FR-140`（轻后端与使用统计）、`NFR-007`（可访问公共网站）。
- `FR-001` / `IA-001`（全局 AppShell：Logo + 主导航 + 全局搜索 + GitHub 登录态 + 主色"提交模块"CTA）。
- `UI-003`（AppShell / Card / StatBlock / LineChart / Accordion 组件）、`UI-001`（令牌）、`UI-004`（生成 / 还原一致性）。
- 来源真源：`docs/design/generated/IA-013-about-repo.html`（TopNavBar + main 五区块 + footer）。

### route 或 surface ID

- route：`/about`（同一界面同时承担"关于 / 仓库入口 / 帮助"语义；主导航"关于我们"激活态，对应真源 HTML 顶栏 `关于我们` 高亮项）。
- 深链接：`/about#hero` `/about#stats` `/about#privacy` `/about#faq` `/about#audit-links` 锚到各区块。
- surface：桌面 Web 优先（`ASM-016`），主内容 `1280px` 居中容器（`UI-001`）；移动 / 平板按 `UI_RULES.md` 响应式规则单列堆叠。

### 数据需求

- 仅 `PAGE-102` 需要动态数据（平台聚合统计 `ENT-019`，详见 `PAGE-102`）。
- `PAGE-101`/`PAGE-103`/`PAGE-104`/`PAGE-105` 为静态内容（理念文案、说明卡、FAQ 条目、链接目标），可随页面静态发布或来自内容配置，不含任何 PII、不含任何模块 / 用户 / 交换的私有内容（`INV-01`、`INV-04`）。
- AppShell 数据：当前 GitHub 登录态（匿名 / 已登录），由 `IA-001` 提供，本页只读消费（不在本页发起登录逻辑契约，归 `shell-discovery` 模块）。

### 动作

- 区块内动作分别见 `PAGE-101`~`PAGE-105`。
- 整页级：AppShell 导航跳转、全局搜索入口、"提交模块"CTA（跳 `IA-004`，需登录由 AppShell 处理）；锚点平滑滚动到各区块。
- 本页无任何破坏性 / 跨边界动作，无同意门（页面为纯展示，不触发 `NFR-005` 节点）。

### 状态

- `default`：五区块按序渲染。
- `loading`：仅统计区有独立加载态（见 `PAGE-102`）；其余区块静态即时可见。
- `error`：统计区失败时降级（见 `PAGE-102`），其余区块不受影响（区块隔离，`PAGE-100` 不因单区块失败整页报错）。
- 响应式：`<768px` 单列、Hero 图文上下堆叠、统计 StatBlock 1 列、FAQ 满宽（`UI_RULES.md` 响应式规则、`ASM-016`）。

### 校验与错误行为

- 无表单输入，无客户端校验。
- 整页错误边界：任一动态区块（`PAGE-102`）失败被局部捕获并降级，不冒泡为整页错误（`UI_RULES.md` 组件行为：状态用文字 + 图标，不仅靠颜色，`NFR-007`）。
- 路由级 404 / 非法锚点：回落到 `/about` 顶部。

### 遥测或分析

- 本页是 `FR-140` / `FLOW-008` 聚合统计的**展示端**，自身只读取轻后端聚合指标（`ENT-019`），不在本页采集任何用户级行为埋点用于 PII（`INV-09`、`NFR-001`）。
- 若未来记录页面访问量，必须以聚合计数形式并入 `ENT-019` 口径、不含 PII（标注假设 `ASM-058`，见返回清单）。

### 验收检查

- `/about` 公开匿名可访问，无需登录即可看到全部静态区块（`IA_SPEC.md` 权限表"公开但匿名可看"）。
- 五区块按 Hero → 统计 → 隐私信任 → FAQ → 可审计链接顺序渲染，与真源 HTML 结构一致。
- 主色 `#017A6E`、仅 Material Symbols Outlined 图标、卡片白底 + `#E7EAEE` 描边 + 12px 圆角（`UI-004` 第 5 条生成后一致性检查）。

- 键盘可导航、桌面 / 移动可读（`NFR-007`，前端验证阶段以浏览器证据核验）。

## PAGE-102 · 平台统计区（StatBlock + 月度活跃折线图）

### 页面用途

- 兑现 `FR-140`（平台使用统计）与 `FLOW-008`（轻后端聚合统计）的**公开展示**：以四个 `StatBlock`（模块总数 / 交换总数 / 活跃用户 / 隐私门通过率）+ 月度活跃趋势 `LineChart` 呈现平台健康度。
- 关键约束：所有数字来自 `ENT-019` UsageStat，**仅聚合、不含 PII**（`DEC-011`、`INV-09`、`NFR-001`）。
- 真源对应 HTML `<!-- Platform Stats -->`：四 StatBlock（`inventory_2`/`swap_horiz`/`group`/`verified_user` 图标，`1,842`/`12,857`/`2,196`/`98.6%`）+ "平台月度活跃趋势"折线图（主色 `#017A6E` 线 + 渐变面积，图例"活跃交换对"，X 轴 1–11 月）。

### 上游需求链接

- `FR-140`、`FLOW-008`、`ENT-019`、`DEC-011`、`NFR-001`、`INV-09`。
- `UI-003`：`StatBlock`（大数字 `--text-stat` + 标签 + Material Symbols Outlined 图标）、`LineChart`（线用 `--color-accent` / 真源用主色，登记一致性差异见验收）。
- `NFR-007`：统计数字有文字标签、图表有可读文字摘要。

### route 或 surface ID

- `/about#stats`（`PAGE-100` 第二区块）。

### 数据需求

- 来自轻后端聚合接口（服务契约阶段定义 `API-*`，对应 `IA-012` 的 `GET /api/stats`）的 `ENT-019` 指标：
  - `modules_total`（模块总数，示例 `1,842`）
  - `exchanges_total`（交换总数，示例 `12,857`）
  - `active_users`（活跃用户，示例 `2,196`；口径 = 时间窗内活跃，聚合计数）
  - `privacy_gate_pass_rate`（隐私门通过率，示例 `98.6%`；= pass / 总提交，聚合派生，关联 `ENT-005`）
  - `monthly_active_series`（月度活跃趋势序列，示例"活跃交换对"按月，1–11 月）
- 每个指标随附**口径 / 时间窗**元信息（`ENT-019` 属性"口径、时间窗"），供图表可读摘要使用。
- 硬约束：响应体绝不含任何 PII、任何单用户 / 单模块 / 单交换可识别明细（`INV-09`、`INV-04`）；只接受聚合标量与时间序列。

### 动作

- 无交互写动作；图表可悬停看点值（聚合值，非个体）。
- 可选"按时间窗切换"（如近 6 / 12 月）——若实现须仍走聚合接口、不引入个体明细（标注假设 `ASM-059`，见返回）。

### 状态

- `loading`：StatBlock 与图表显示骨架 skeleton（真源有数字递增动画，属展示态）。
- `default`：四 StatBlock + 折线图渲染。
- `empty`（统计无数据 / 平台早期）：StatBlock 显示"—"或"暂无数据"文字、图表显示空摘要（对应 `UI_RULES.md` `EmptyState`；真源提示"统计无数据"）。
- `error`：聚合接口失败→区块内显示"统计暂不可用，请稍后重试"（文字 + 图标，不仅颜色），不影响其余区块（`PAGE-100` 区块隔离）。

### 校验与错误行为

- 客户端只渲染服务端给定的聚合值，不做任何反推个体的计算。
- 若返回字段缺失 / 非数值→该 StatBlock 单独降级为 `empty`，其余正常。
- 百分比（隐私门通过率）渲染范围校验 `0–100%`，越界视为数据异常→该卡 `empty` + 上报（聚合层面）。

### 遥测或分析

- 本区块是 `FR-140` / `FLOW-008` 的**消费端 / 展示端**；不产生新的个体埋点。
- 显示的指标即平台对外披露的聚合口径；任何新增展示指标必须先在 `ENT-019` / 服务契约登记聚合口径（`NFR-003` 可追溯）。

### 验收检查

- 四 StatBlock 文案 = 模块总数 / 交换总数 / 活跃用户 / 隐私门通过率，数字有文字标签（非仅大数字，`NFR-007`）。
- 折线图提供可读文字摘要（如"近 11 个月活跃交换对整体上升"），供读屏（`NFR-007`）。
- 渲染数据全部为聚合标量 / 序列，无任何 PII 或个体明细（`INV-09`、`INV-04`）——以接口契约 + 前端断言核验。
- `loading`/`empty`/`error` 三态可达且区块隔离。
- 折线 / 图表色彩遵 `UI-001`（真源用主色 `#017A6E`；规范 `LineChart` 默认 `--color-accent`，二者均为令牌内色，登记为可接受一致性差异，组件规格阶段定稿统一）。

## PAGE-103 · 隐私与信任说明卡

### 页面用途

- 用一组说明卡把平台的隐私边界与信任形成机制讲清，强化 `NFR-001`（数据最小化）、`NFR-005`（人类同意门）、`NFR-004`（可审计）的对外解释。
- 真源对应 HTML `<!-- Privacy & Trust -->`：三张 `Card`——"不托管原始内容"（`database_off`）、"同意优先"（`lock_open`）、"信任如何积累"（`vitals`）。

### 上游需求链接

- `NFR-001`（不托管原始内容；`INV-01`）、`NFR-005`（同意优先；`INV-08`）、`NFR-004`（可审计）。
- 信任解释追溯 `FR-050`/`FR-060`/`ENT-011`（信任由交换历史 / 反馈质量 / 验证 / 举报派生且可解释），`INV-10`（参与方反馈权重高于社交信号）。
- `UI-003`：`Card`（白底 + 描边 + 圆角 + 图标圆容器）。

### route 或 surface ID

- `/about#privacy`（`PAGE-100` 第三区块）。

### 数据需求

- 静态文案三张卡。卡片内容必须与产品真源一致，**不得照搬真源 HTML 中漂移的技术主张**：
  - 卡 1"不托管原始内容"：表述平台只存清单与公开关系、原始内容留在所有者本地（`INV-01`、`NFR-001`）。
  - 卡 2"同意优先"：表述生成 / 提交 / 联系 / 交换前均有所有者明确同意门（`NFR-005`、`INV-08`），私下交付走平台外通道（默认 GitHub 私有仓库，`ASM-007`）。**不得**声称"端到端加密 E2EE"——产品规格未承诺该机制。
  - 卡 3"信任如何积累"：表述信任由交换历史 / 结构化反馈 / GitHub 验证 / 举报派生、页面可解释、参与方反馈权重更高（`FR-050`、`ENT-011`、`INV-10`）。**不得**声称"链上 / 不可篡改链上记录"——产品无区块链机制（见整体风险）。

### 动作

- 静态卡，无主动作；卡可整体作为指向更详细说明的入口（可选链到 `PAGE-105` 的隐私模型）。
- 无同意门（仅解释，不执行跨边界动作）。

### 状态

- `default`：三卡渲染，hover 描边变主色（真源行为）。
- 响应式：`md` 以下 1 列。
