## PAGE-001 — 全局站点外壳 AppShell

### Page purpose

全站持久外壳：在每个页面顶部提供一致的 Logo、水平主导航、全局搜索、GitHub 登录态、以及主色"提交模块"CTA；并承载全局加载/错误/离线降级与未登录↔登录态差异。对应视觉真源 `know-share-website-style-v1.png` 顶栏（左 Logo「K Know-share」+ 主导航「发现 / 交换记录 / 信任网络 / Agent 技能 / 仓库」+ 右 全局搜索框 + 「GitHub 登录」+ 绿色「提交模块」）。浏览公开页无需登录（`FR-001`、`IA_SPEC` 导航模型、权限敏感界面「公开但匿名可看」）。

### Upstream requirement links

- `IA-001`（全局站点外壳）、导航模型与权限敏感界面表（`IA_SPEC.md`）
- `FR-001`（公开站点外壳与身份：全局导航、GitHub 登录状态、全局搜索入口、提交入口；浏览公开页无需登录）
- `FLOW-007`（身份与同意：浏览公开页无需登录；发起交换/提交/社交需 GitHub 身份）
- `DEC-006`（GitHub 为规范身份与验证来源）、`NFR-006`（抗滥用：写动作前身份核查）、`NFR-005`（同意门）
- `NFR-007`（无障碍：键盘可导航、桌面/移动可读）
- `UI-003` 组件 `AppShell`/`SearchBar`/`PrimaryButton`/`SecondaryButton`/`StatusPill`、`UI-002` 图标族、`UI-001` 令牌
- `ENT-001`（User：GitHub 身份、GitHub Verified）、`ENT-017`（Notification：登录态通知入口）

### Route or surface ID

- Surface `IA-001`；非独立路由，作为持久布局包裹所有页面（含 `PAGE-002`/`PAGE-003`）。
- 内含导航目标（路由在各自模块页面规格定义，本规格只锚定外壳出口）：
  - 公共主导航：发现 `/`(`IA-002`)、交换记录 `/exchanges`(`IA-005`)、信任网络 `/trust`(`IA-007`)、Agent 技能 `/skills`(`IA-008`)、仓库/帮助 `/about`(`IA-013`)。
  - 右侧常驻：全局搜索框（路由 `/search?q=` → `PAGE-003`）、主色「提交模块」CTA(`/submit`，`IA-004`)、GitHub 登录态(`IA-001`)。
  - 已登录增项：个人中心 `/me`(`IA-009`)、通知 `/notifications`(`IA-010`，带未读计数徽标)、设置 `/settings`(`IA-014`，重定向到 `/settings/contact`)，头像菜单展开；具管理员角色时显示审核控制台 `/admin`(`IA-011`)。
- Logo 点击回到 `/`(`IA-002`)。

### Data required

- 当前会话身份（`ENT-001`）：是否登录、GitHub 用户名/头像、`GitHub Verified` 标记、是否管理员角色（决定是否显示审核入口）。来源：轻后端会话端点（`FR-140`、`DEC-011`）。
- 未读通知计数（`ENT-017`）：用于通知图标徽标（仅登录态）。
- 主导航项静态配置（IA 决定，非用户数据）。
- 全局搜索的输入查询字符串（提交时传递给 `PAGE-003`）。
- 不取用任何私有内容或 PII 列表（`INV-01`/`INV-04`）；头像/用户名为 GitHub 公开身份，非 `ENT-008` 联系方式（`DEC-010`：联系方式默认私密、绝不进外壳）。

### Actions

- 点击 Logo → 回首屏 `/`。
- 点击主导航项 → 路由到对应 IA 界面。
- 在 SearchBar 输入并提交（Enter 或点击搜索图标）→ 路由 `/search?q=<query>`(`PAGE-003`)。
- 点击「GitHub 登录」（未登录态）→ 发起 GitHub OAuth 登录（`DEC-006`、`FLOW-007`）；成功后回到发起页。
- 点击「提交模块」CTA → 路由 `/submit`(`IA-004`)；未登录时先引导 GitHub 登录（提交需登录 + 同意门，`NFR-005`、`IA_SPEC` 权限表），不在外壳内绕过同意。
- 点击通知图标（登录态）→ 路由 `/notifications`(`IA-010`)。
- 点击头像菜单（登录态）→ 展开：个人中心/设置/（管理员）审核控制台/退出登录。
- 退出登录 → 清除会话、回到 `/` 匿名态。

### States

- **匿名（未登录）态**：右侧显示「GitHub 登录」(SecondaryButton/Octocat 品牌图标) + 「提交模块」(PrimaryButton)；无通知图标、无头像菜单；主导航仅公共项。
- **登录态**：右侧显示通知图标（含未读徽标）+ 头像菜单（含用户名、可含 `GitHub Verified` StatusPill）+ 「提交模块」CTA；主导航增个人项入口（移动端折叠进头像/汉堡菜单）。
- **管理员登录态**：在登录态基础上，头像菜单内出现「审核控制台」入口。
- **加载态**：身份/通知计数加载中→头像区与通知徽标显示骨架占位；外壳结构与导航即时可用（不阻塞浏览）。
- **错误/降级态**：会话端点失败 → 降级为匿名态外壳（仍可浏览公开页，符合 `FR-001` 匿名可看），不弹断页错误；通知计数失败 → 隐藏徽标不报错。
- **离线/后端不可用**：导航与搜索框可见但提交/登录动作禁用并提示「服务暂不可用」（`ASM-019`）。
- **响应式态**：`≥1280px` 完整水平导航；`768–1279px` 次级项折叠；`<768px` 主导航收进汉堡菜单、搜索折叠为图标展开（`UI_RULES` 响应式规则、`ASM-016`）。

### Validation and error behavior

- 搜索框提交空串：不路由，保持当前页（或聚焦输入），无报错。
- 搜索查询长度上限（建议 ≤ 200 字符，`ASM-018`）：超长截断并提示。
- 「提交模块」/写动作在未登录时：拦截并引导登录，而非静默失败（`NFR-005`、`NFR-006`）。
- GitHub 登录失败/取消：回到发起页匿名态，显示非阻断提示「登录未完成」。
- 会话过期：下一次需身份的动作触发重新登录；浏览态不打断。
- 所有状态/角色区分同时有文字与图标，不靠颜色单独传达（`NFR-007`、`UI_RULES` 组件行为规则）。

### Telemetry or analytics

- 仅聚合、不含 PII（`FR-140`、`DEC-011`、`INV-09`、`NFR-001`）。
- 外壳级聚合事件（计数，不绑定个人）：导航项点击分布、全局搜索发起次数、登录发起/成功次数、提交 CTA 点击次数。用于平台使用统计（`FLOW-008` → 喂给 `IA-013`/发现页底部 strip 的 `ENT-019`）。
- 不记录搜索词与用户身份的关联明细（避免 PII 关联，`INV-09`、`ASM-020`）。

### Acceptance checks

- 匿名用户可见并使用全部公共主导航与全局搜索，无需登录即可浏览（`FR-001`、`IA_SPEC` 权限表「公开但匿名可看」）。
- 「提交模块」与任何写动作在未登录时被引导至 GitHub 登录，不存在绕过同意门的路径（`NFR-005`、`NFR-006`）。
- 外壳从不展示任何 `ENT-008` 联系方式或私有内容（`INV-01`/`INV-04`、`DEC-010`）。
- 顶栏视觉与 `know-share-website-style-v1.png` 一致：主色 `#017A6E` CTA、Logo、仅 Material Symbols Outlined 图标 + GitHub Octocat 例外（`UI-001`/`UI-002`/`DEC-012`）。
- 键盘可遍历全部外壳交互元素，焦点态可见；图标按钮有 `aria-label`（`NFR-007`）。
- 后端会话失败时降级为可浏览的匿名外壳，不出现整页错误。

### Upstream requirement links

- `IA-002`（发现/注册表：可搜索/筛选/排序的模块列表、热门主题、平台统计）、内容层级「首屏 = 发现」
- `FR-010`（发现/注册表：可搜索/筛选/排序的模块卡片、热门主题、平台统计、请求交换入口；不暴露私有内容）
- `FR-140`（底部平台聚合统计）、`FLOW-008`（平台使用统计）、`FLOW-002`（发现与评估）
- `DEC-009`（互惠可选 → 卡片 CTA 为「请求交换」，不强制配对）、`DEC-011`（聚合统计）
- `NFR-001`/`INV-01`/`INV-04`（不暴露私有内容/私有 URL）、`INV-09`（统计仅聚合不含 PII）、`NFR-007`（无障碍）
- `ENT-003`（KnowledgeModule）、`ENT-004`（Manifest 脱敏摘要/来源统计/主题/新鲜度/信任信号）、`ENT-020`（Topic/Tag）、`ENT-019`（UsageStat）、`ENT-011`（派生信任级别，卡片信任信号）
- `UI-003` 组件 `ModuleCard`/`SearchBar`/`StatBlock`/`StatusPill`/`Card`/`IconChip`/`EmptyState`/`SecondaryButton`/`PrimaryButton`、`UI-001` 令牌（卡片网格 ≥1280px 3–4 列）

### Route or surface ID

- 路由 `/`（首屏），等价别名 `/discover`（surface `IA-002`）。
- 区域：①标题/价值主张 Hero；②筛选 + 排序控制行 + 热门主题标签（`ENT-020`）；③`ModuleCard` 卡片网格（主区）；④底部平台统计 strip（`StatBlock` 网格 + 社区/为什么选我们的说明卡）。
- 进入点：直达 `/`、AppShell 导航「发现」、Logo、深链接（带筛选参数 `/?topic=&sort=&type=`）。
- 出口：卡片标题/卡片体 → 模块详情 `/modules/:id`(`IA-003`、`FR-020`)；卡片「请求交换」→ 交换请求流（`IA-006`/`FR-040`，需登录）；卡片作者/来源 → 用户档案(`IA-007`)；主题标签 → 以该主题筛选；底部统计「了解更多」→ `IA-013`。

### Data required

- 模块卡片列表（`ENT-003` + `ENT-004`），每卡：模块标题、类型（`ENT-003.type`）、脱敏摘要（`ENT-004` 摘要，**绝不含原文/私有 URL**，`INV-01`/`INV-04`）、来源统计（来源类别/条目数等聚合，`ENT-004`）、主题标签（`ENT-020`）、新鲜度（派生更新时间）、信任信号（`ENT-011` 派生信任级别 + 可含 `GitHub Verified`）、社交计数（收藏/交换数派生，`FR-070`）、所有者 GitHub 身份（公开，非 PII）。
- 筛选维度：模块类型、主题标签（`ENT-020`）、信任级别、可含「是否 Verified」；排序维度：相关度/最新/最热门/信任分（具体集合见 `ASM-017`）。
- 热门主题标签集合（`ENT-020`，按使用频次的聚合）。
- 平台聚合统计（`ENT-019`）：模块总数、交换总数、活跃用户数、隐私门通过率（与真源 `1,842 / 12,857 / 2,196 / 98.6%` 同口径，来源轻后端 `FR-140`、`FLOW-008`）。
- 分页/游标或「加载更多」状态。
- 当前用户登录态（来自 `PAGE-001`）：决定「请求交换」是直接进入还是先引导登录。
- 不取用：任何原始知识内容、私有仓库 URL、联系方式（`INV-01`/`INV-04`/`DEC-010`）。

### Actions

- 在页内/外壳搜索框输入关键字 → 过滤/路由到结果（与 `PAGE-003` 一致的查询）。
- 选择筛选条件（类型/主题/信任级别）→ 即时过滤列表并更新 URL 查询参数（可分享/可深链）。
- 切换排序 → 重排列表并更新 URL。
- 点击热门主题标签 → 以该主题筛选。
- 点击模块卡（标题/卡体）→ 进入模块详情 `/modules/:id`(`IA-003`)。
- 点击卡片「请求交换」CTA → 登录态进入交换请求流（`FR-040`、`IA-006`）；未登录先引导 GitHub 登录（`NFR-005`/`NFR-006`），不在发现页发起任何写操作。
- 点击卡片「收藏/认可」轻量社交动作（若展示）→ 登录态写入 `ENT-013`，受唯一性约束与速率限制（`FR-070`、`INV-07`、`NFR-006`）；未登录引导登录。
- 「加载更多」/翻页 → 拉取下一批模块。
- 底部统计 strip「了解更多/关于」→ 路由 `IA-013`。

### States

- **默认（有数据）**：标题 + 筛选行 + 卡片网格（≥1280px 3–4 列 / 平板 2 列 / 移动 1 列）+ 底部统计 strip，匹配真源布局。
- **加载态**：卡片区显示骨架卡占位；统计 strip 显示数字骨架；筛选/排序控件即时可用。
- **空注册表/无任何模块**：`EmptyState`（居中 Material Symbols 图标 + 文案「还没有公开模块」+ 主 CTA「提交首个模块」→ `IA-004`），对齐 `IA_SPEC` 空状态。
- **筛选/搜索无结果**：`EmptyState` 文案「没有匹配的模块」+ 「清除筛选」次按钮 + 调整建议，保留筛选控件（`IA_SPEC` 空状态）。
- **错误态**：模块列表加载失败 → 行内错误卡 + 「重试」；统计 strip 失败 → 该 strip 静默隐藏或显示「统计暂不可用」，不阻断卡片浏览。
- **登录差异**：匿名态卡片 CTA 仍可见但点击引导登录；登录态可直接进入交换/社交动作。
- **响应式**：网格列数随断点变化；筛选行在移动端折叠为「筛选」抽屉/下拉（`ASM-016`）。

### Validation and error behavior

- 卡片渲染前对清单字段做安全校验：若某卡数据疑似含私有内容/私有 URL（违反 `INV-01`/`INV-04`），不渲染该字段并按降级显示（兜底不外泄，契约层应已保证，此为前端二次防线，`ASM-020`）。
- 筛选/排序参数非法（URL 被篡改）→ 回退到默认筛选/排序，不报错。
- 搜索查询过长（与 `PAGE-001` 一致上限，`ASM-018`）→ 截断。
- 统计数字缺失/为 0：显示 `0` 或「—」，不显示占位假数据。
- 写动作（请求交换/收藏）失败：行内提示 + 可重试；速率超限给出明确「操作过于频繁」提示（`NFR-006`）。
- 状态/信任级别用 `StatusPill` + 文字 + 图标，不仅靠颜色（`NFR-007`）。

### Telemetry or analytics

- 仅聚合、不含 PII（`FR-140`/`DEC-011`/`INV-09`/`NFR-001`）。
- 发现面聚合事件（计数）：搜索发起、各筛选/排序使用频次、热门主题点击、卡片曝光/点击（聚合，不绑定个人）、「请求交换」CTA 点击（漏斗入口）、「加载更多」次数。
- 底部统计 strip 直接呈现 `ENT-019` 聚合指标（模块/交换/活跃用户/隐私门通过率），由 `FLOW-008` 轻后端计算（`FR-140`）。
- 不记录"用户 X 看了模块 Y"的可识别明细；曝光/点击仅做去标识化聚合（`INV-09`、`ASM-020`）。

### Acceptance checks

- 用户仅凭卡片清单元数据（脱敏摘要/来源统计/主题/信任信号）即可判断是否请求交换，无需也无法看到原始内容（`FR-010`、`PRODUCT_SPEC` 验收「发现可用」、`INV-01`/`INV-04`）。
- 搜索 + 筛选 + 排序可组合工作并反映在可分享的 URL 上；无结果时给出 `EmptyState` 与清除筛选。
- 底部平台统计为聚合指标、不含 PII，且与真源口径一致（模块/交换/活跃用户/隐私门通过率）（`FR-140`、`INV-09`、`FLOW-008`）。
- 卡片「请求交换」对匿名用户引导登录、对登录用户进入交换流（`FR-040`、`NFR-005`）；互惠为可选不强制（`DEC-009`）。
- 视觉与 `know-share-website-style-v1.png` 一致：`ModuleCard`/`StatBlock` 规范、主色 CTA、仅 Material Symbols Outlined 图标（`UI-001`/`UI-002`/`UI-003`/`DEC-012`）。
- 公开面绝不出现仓库内容或私有 URL（人工 + API 输出抽查，`INV-04`、`PRODUCT_SPEC` 成功信号）。
- 键盘可达卡片、筛选、排序、CTA；信任/状态非仅颜色区分（`NFR-007`）。

## PAGE-003 — 全局搜索结果 surface

### Page purpose

承接 AppShell 全局搜索框的提交，给出跨实体的搜索结果落地面。按 `FR-001`「全局搜索在相应规格就绪后路由到模块、主题、用户或交换」，本 surface 把查询解析为分类结果（模块/主题/用户/交换记录），其中模块结果复用发现页的 `ModuleCard` 呈现并保持"不暴露私有内容"的同一边界。它是外壳能力（`IA-001`）在发现语境（`IA-002`）下的结果面，避免把搜索结果硬塞进发现页主列表。

### Upstream requirement links

- `IA-001`（全局搜索入口）× `IA-002`（注册表呈现）、`IA_SPEC` 导航模型「全局搜索」条
- `FR-001`（全局搜索路由到模块/主题/用户/交换）、`FR-010`（模块结果呈现）
- `NFR-001`/`INV-01`/`INV-04`（结果不含私有内容/私有 URL）、`NFR-002`（结构化、agent 可读边界一致）、`NFR-007`（无障碍）
- `ENT-003`/`ENT-004`（模块结果）、`ENT-020`（主题结果）、`ENT-001`（用户结果，公开身份）、`ENT-007`（交换记录结果，脱敏台账）
- `UI-003` 组件 `SearchBar`/`ModuleCard`/`ListRow`/`StatusPill`/`EmptyState`/`Card`

### Route or surface ID

- 路由 `/search?q=<query>`（可含 `&type=` 限定分类），由 `PAGE-001` 的 SearchBar 提交进入。
- 结果分组：模块（→ `IA-003`）、主题（→ 以该主题筛选的 `PAGE-002`）、用户（→ `IA-007`）、交换记录（→ `IA-005`/`IA-006` 脱敏台账）。
- 出口同各结果类型对应 IA；顶部保留 SearchBar 以便改词重搜。

### Data required

- 解析后的查询字符串与可选分类过滤。
- 分类结果集合：
  - 模块结果（`ENT-003`+`ENT-004`，字段同 `PAGE-002` 卡片，脱敏边界一致）。
  - 主题结果（`ENT-020` 名称 + 关联模块数）。
  - 用户结果（`ENT-001` 公开身份：GitHub 用户名/头像/`GitHub Verified`/信用分派生；非 PII、非 `ENT-008`）。
  - 交换记录结果（`ENT-007` 脱敏台账行：方向/状态/时间，不含私有通道内容，`INV-04`）。
- 各分类结果计数；分页/加载更多状态。
- 当前登录态（决定结果项上写动作的引导）。
- 不取用：原始内容、私有 URL、联系方式（`INV-01`/`INV-04`/`DEC-010`）。

### Actions

- 在顶部 SearchBar 改词重搜 → 更新 `?q=`。
- 切换分类 Tab（全部/模块/主题/用户/交换）→ 更新结果分组与 `?type=`。
- 点击模块结果 → `IA-003`；主题结果 → 以该主题筛选的 `/?topic=`(`PAGE-002`)；用户结果 → `IA-007`；交换记录结果 → `IA-005`/`IA-006`。
- 「加载更多」→ 拉取该分类下一批。

### States

- **默认（有结果）**：分类 Tab + 各组结果列表（模块用 `ModuleCard`，其余用 `ListRow`）。
- **加载态**：结果区骨架占位。
- **无结果**：`EmptyState`「没有与 “<query>” 匹配的结果」+ 建议（检查拼写/换关键词/去发现页浏览 → `PAGE-002`）。
- **空查询**：若直接访问 `/search` 无 `q` → 重定向/引导回发现页或显示提示输入。
- **错误态**：搜索后端失败 → 错误提示 + 「重试」；不暴露后端细节。
- **登录差异**：结果项上的写动作（如对模块「请求交换/收藏」）匿名引导登录。
- **响应式**：分类 Tab 在窄屏可横向滚动；模块结果网格随断点变列（`ASM-016`）。

### Validation and error behavior

- 空 `q`：不发起空搜索，引导输入或回发现页。
- 查询长度上限（`ASM-018`）：超长截断并提示。
- 任一结果项疑似含私有内容/私有 URL：前端二次防线不渲染该字段（`INV-01`/`INV-04`、`ASM-020`）。
- 非法 `type` 参数：回退「全部」。
- 状态/类型非仅颜色区分（`NFR-007`）。

### Telemetry or analytics

- 仅聚合、不含 PII（`FR-140`/`DEC-011`/`INV-09`）。
- 聚合事件：搜索发起次数、各分类结果点击分布、无结果率（用于改进发现）。
- 不持久化「查询词 ↔ 用户身份」可识别关联（`INV-09`、`ASM-020`）。

### Acceptance checks

- 全局搜索可把查询路由到模块/主题/用户/交换四类结果，并各自跳转到正确 IA（`FR-001`）。
- 模块结果与发现页保持同一脱敏边界，绝不暴露私有内容/私有 URL（`INV-01`/`INV-04`）。
- 无结果/空查询有明确 `EmptyState` 与引导，不报裸错误。
- 结果项写动作对匿名用户引导登录（`NFR-005`/`NFR-006`）。
- 键盘可达 SearchBar、分类 Tab 与结果项；无障碍达标（`NFR-007`）。
