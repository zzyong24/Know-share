## 摘要

## PAGE-040 用户信任档案页

### Page purpose

公开声誉面：任意访客（含未登录、含 agent）在此查看某位 `User`（`ENT-001`）的可解释信任画像——GitHub 身份与 `GitHub Verified` 状态、信任分与等级（大圆环 `824 / 1000`）、声誉趋势折线、获得的徽章墙、反馈质量摘要、已发布模块、交换历史、社交认可/关注者。这是消费方 agent 与人在「是否信任此人、是否值得发起交换」上的核心判断面，也是 `HARD-03`「信任可解释」落到界面的主承载页。对应设计真源 `know-share-trust-profile.png` 的整屏布局（左身份卡 + 右信任分圆环 + 指标 StatBlock 行 + 趋势折线 + 已发布模块/交换历史/徽章/反馈质量多列 + 底部认可与近期反馈）。

### Upstream requirement links

- `IA-007`（信任网络 / 用户信任档案，声誉面）— 顶层界面定义、主要实体 `ENT-001/011/012`、来源 `FR-060/FR-050`。
- `FR-060` 用户信任档案：公开贡献者档案（身份、声誉指标、已发布模块、交换历史、认可、反馈质量）。
- `FR-050` 结构化反馈与信任：可见信任级别、徽章；参与方反馈权重高于社交信号。
- `HARD-03`：信任级别/信用分由交换历史/反馈质量/验证/举报得出，**页面可解释**；权重见 `INV-10`。
- `ENT-001` User（GitHub 身份、Verified、领域标签、加入日期、信用分(派生)、徽章(派生)）。
- `ENT-011` TrustProfile（派生：用户信用分、分数趋势、解释）。
- `ENT-012` Badge（类型、授予条件）。
- `ENT-003` KnowledgeModule（已发布模块卡，复用 `ModuleCard`，只读摘要面）。
- `ENT-007` Exchange（交换历史，只读、不含私有内容）。
- `ENT-013` SocialSignal（认可 / 关注者，权重低于参与方反馈）。
- `INV-04` 公开记录与 API 输出不含私有内容或私有 URL（声誉面公开匿名可看，`IA_SPEC.md`「公开但匿名可看」）。
- `INV-10` 交换参与方反馈对信任的权重高于社交信号。
- `DEC-006` GitHub 规范身份；`DEC-007` 无经济模型（本页不得出现任何付费/打赏/排名付费提升元素）。
- UI：`UI-003` 组件 `TrustBadge`、`StatBlock`、`LineChart(accent)`、`ListRow`、`ModuleCard`、`StatusPill`、`Card`、`AppShell`；`UI-001` 令牌；`UI-002` Material Symbols Outlined。

### Route or surface ID

- Route: `/u/:githubLogin`（规范公开档案路由，键 = GitHub login，呼应 `DEC-006`；例 `/u/zyongzhu24`）。
- 别名/兼容: `/profile/:githubLogin` 可作 301 跳转到规范路由（`ASM-038`）。
- 入口（`IA_SPEC.md` 导航模型「二级进入」）：主导航「信任网络」、`ModuleCard` 上的作者链接、交换记录(`IA-005`)/交换详情(`IA-006`) 中的用户、个人中心(`IA-009`)「查看我的公开档案」、全局搜索用户结果。
- 访问：公开匿名可看（`INV-04`）；登录态额外解锁「关注 / 认可」等社交动作（见 Actions）。
- 自看视角：本人查看自己档案时，提供「编辑档案 / 联系方式设置」入口跳 `IA-014`（仅链接，不在本页内编辑）。

### Data required

只读聚合，全部来自公开/派生数据，**不含 PII、不含私有内容**（`INV-04`）：

- **身份头（`ENT-001`）**：头像、显示名、GitHub login（@handle）、`GitHub Verified` 徽（`StatusPill` success + Octocat，`DEC-006`）、一句话简介、领域标签（`ENT-020` 引用）、加入日期、外部 GitHub 链接。联系方式默认私密、**不在此页展示**（除非用户显式 opt-in 为 public 的 handle，如公开的 GitHub —— 仅展示已 public 的，`INV-03`/`DEC-010`）。
- **信任分圆环（`ENT-011` 派生）**：当前信用分（如 `824`）、满分基准（`1000`，`ASM-037`）、信任等级文案（如「资深贡献者 / Trusted」）、`TrustBadge`（盾形 + 等级）。圆环为环形进度（主色分段，`UI-003` DonutChart/环形）。
- **可解释拆解（`HARD-03` 必备）**：信任分的派生构成分项——交换历史贡献、反馈质量贡献、GitHub 验证贡献、举报/扣分项——每项的标签、占比/分值方向、一句话说明（数据来自 `ENT-011.解释`）。承载在「信任分如何形成」区，详细弹层走 PAGE-041。
- **声誉趋势（`ENT-011.分数趋势`）**：时间序列点（近 N 期信用分），渲染为 `LineChart`（`--color-accent`，`UI-003`）；含可读文字摘要（无障碍，`NFR-007`）。
- **关键指标 StatBlock 行**（设计图顶部一排，派生计数）：贡献模块数、累计交换次数、被收藏数（`ENT-013`）、清单一致性率（如 `94%`，来自反馈维度聚合）、平均反馈评分（如 `4.7`）、社区排名（可选，`ASM-039`）。每项 `StatBlock`（大数字 + 标签 + Material Symbols Outlined 图标 + 可选趋势）。
- **已发布模块（`ENT-003`）**：该用户 `Published` 状态模块列表，复用 `ModuleCard`（标题、脱敏摘要、主题标签、信任级别、交换/收藏计数、「查看详情」链接到 `IA-003`）。仅公开摘要面，**不含原始内容**（`INV-01/04`）。
- **交换历史（`ENT-007`）**：该用户参与并已完成/公开的交换记录行（`ListRow`）：对方 @handle（链接到其档案）、方向（请求/被请求，Material Symbols Outlined 方向图标）、涉及模块名（脱敏标题）、状态 `StatusPill`（`Completed`/`Closed` 等公开态）、时间。**不含私有通道引用、不含联系方式、不含仓库内容**（`INV-04`）。
- **徽章墙（`ENT-012`）**：已获徽章网格，每枚 `TrustBadge`/`IconChip`（图标 + 名称 + 授予条件 tooltip，如「完成 10 次零争议交换」）。
- **反馈质量（`ENT-010` 聚合）**：作为被评价方收到的结构化反馈维度均值（清单一致性、隐私边界、结构清晰度、有用性、再次交换意愿）以小型条/评分展示 + 近期反馈摘要片段（公开文本、作者 @handle、关联交换链接）。须标注「来自实际交换参与方」，与社交认可视觉分区（体现 `INV-10`）。
- **社交认可 / 关注者（`ENT-013`）**：认可者/关注者头像墙 + 计数；明确低权重、视觉次于反馈质量区。

> 数据来源边界：本页所有数据经 `IA-012` 公开读 API 同源口径（`INV-04` 零私有泄露），与匿名访客所见一致。

### Actions

- **查看派生明细**：点信任分圆环或「信任分如何形成」→ 打开 PAGE-041（解释弹层/抽屉）。
- **跳已发布模块详情**：`ModuleCard` →`/modules/:id`（`IA-003`）。
- **跳交换详情**：交换历史行 → `IA-006`（仅本人或交换相关方可见全量；公开访客见脱敏台账视图，`INV-04`）。
- **跳对方档案**：交换历史 / 认可者头像 → 该用户 PAGE-040。
- **关注 / 取消关注**（需登录，`ENT-013`，`FLOW-007`/`NFR-006`）：受唯一性约束与速率限制；写 `AuditLog`（`INV-11`）。
- **认可（endorse）**（需登录，`ENT-013`）：低权重社交信号，唯一性约束 + 速率限制 + 审计（`NFR-006`/`INV-11`）；**明确不影响信任分权重高于反馈**（`INV-10`）。
- **举报该用户**（需登录，`ENT-014`）→ 触发举报流（`FLOW-005`，由 admin 模块处理；本页仅提交入口）。
- **本人自看**：「编辑档案 / 联系方式设置」→ `IA-014`（不在本页内联编辑）。
- **请求交换 CTA**：在某 `ModuleCard` 上提供「请求交换」入口（主色 `PrimaryButton`），跳模块详情发起流（`FR-040`，归 exchange 模块；本页仅引导）。

### States

- **加载**：身份头 + 圆环 + StatBlock + 各列表用 skeleton（`UI-003` loading）。
- **就绪（默认）**：完整渲染设计图所有区。
- **新用户/无历史**（`IA_SPEC.md` 空状态「无反馈/无信任历史」）：信任分显示基础分 + 「信任如何随交换积累」说明 `EmptyState`；趋势折线显示「暂无足够数据」；已发布模块为空时引导（若为本人）走提交向导（`IA-004`）。
- **未验证 GitHub**：不显示 `GitHub Verified` 徽，代之以中性「未验证」`StatusPill`，并降低信任分中验证贡献项（在 PAGE-041 解释）。
- **被举报/受限**（`Flagged`/处罚态）：若该用户因审核处罚被降权，页面显示中性状态提示（不泄露举报细节，`INV-04`/`INV-11`），信任分拆解中体现扣分项。
- **未登录访客**：所有只读区可见（`INV-04`）；社交动作（关注/认可/举报）显示为「登录后可用」。
- **错误**：用户不存在 → 404 `EmptyState`；数据加载失败 → 错误态 + 重试。

### Validation and error behavior

- **路由校验**：`:githubLogin` 不存在 → 404；存在但未注册平台 → 「该 GitHub 用户尚未加入 Know-share」`EmptyState`（不暴露是否存在的私密信号边界，`ASM-038`）。
- **社交动作约束**（`NFR-006`）：关注/认可需登录；每 (User, target) 至多一次有效认可（类比 `INV-07`）；速率限制触发 → 友好提示「操作过于频繁，请稍后再试」；所有写动作记 `AuditLog`（`INV-11`）。
- **零私有泄露守卫**（`INV-04`）：渲染层与 API 同口径，交换历史/模块卡严禁出现私有仓库 URL、联系方式、原始内容；若数据异常含敏感字段，前端按字段白名单丢弃并上报告警（不渲染）。
- **信任分可解释性硬校验**（`HARD-03`）：信任分展示处必须同时可达其拆解（PAGE-041 入口不可缺失）；若派生解释数据缺失，圆环旁显示「解释生成中」占位而非裸分值。
- **无经济元素守卫**（`DEC-007`）：禁止任何付费/打赏/付费置顶；排名/徽章不得标注价格或可购买。

### Telemetry or analytics

对齐 `FR-140`/`FLOW-008`，仅**聚合、不含 PII**（`INV-09`）：

- 事件（聚合计数，匿名化）：`profile_view`（档案被查看次数，聚合到平台「活跃度」口径，不存查看者身份明细）、`trust_breakdown_open`（信任分拆解打开次数）、`module_card_click`、`exchange_history_click`、`endorse`/`follow`（社交动作量，聚合）。
- 用途：平台使用统计（`ENT-019` UsageStat：用户数/模块数/交换数等聚合），公开展示于 `IA-013`。
- 约束：不记录「谁查看了谁」的可识别映射；遥测口径与 `IA-013` 统计一致（`INV-09`/`DEC-011`）。

### Acceptance checks

- 公开匿名访客可在不登录、不接触任何私有内容的前提下看到完整信任画像（`INV-04`、`IA_SPEC.md` 权限表）。
- 信任分与等级旁**始终可达**其派生拆解（交换历史/反馈质量/GitHub 验证/举报），满足 `HARD-03` 可解释（PAGE-041 入口存在且有效）。
- 反馈质量区明确标注「来自实际交换参与方」，且在视觉与说明上权重高于社交认可区（`INV-10`/`FR-050`）。
- 已发布模块与交换历史均为脱敏只读，绝不出现原始内容、私有 URL、联系方式（`INV-01/04`）。
- 页面无任何付费/打赏/计费/可购买元素（`DEC-007`）。
- 视觉锚定规范设计系统：主色 `#017A6E`、仅 Material Symbols Outlined、`TrustBadge`/`StatBlock`/`LineChart(accent)`/`ListRow`/`ModuleCard` 与 `UI-003` 一致（`DEC-012`）。
- 无障碍（`NFR-007`）：圆环/折线提供文字摘要、状态非仅靠颜色、社交动作键盘可达、徽章 tooltip 可达。
- 遥测仅聚合、不含 PII，且不记录可识别的「谁看了谁」（`INV-09`）。

## PAGE-041 信任分解释（子 surface — 抽屉/弹层）

### Page purpose

把信任分从「一个数字」展开为「**为什么是这个数字**」的可解释明细，直接兑现 `HARD-03`/`NFR-004`（信任解释可审计）。由 PAGE-040 的信任分圆环或「信任分如何形成」入口触发，以右侧抽屉或模态层呈现，不离开档案页上下文。内容是 `ENT-011`（TrustProfile 派生）的解释结构：各贡献维度的来源、方向、权重说明，并显式声明「交换参与方反馈 > 社交信号」的权重规则（`INV-10`）。

### Upstream requirement links

- `HARD-03`（信任可解释，本页是其界面兑现核心）；`FR-050`/`FR-060`。
- `NFR-004` 开源可审计：信任解释可在仓库产物/页面查阅。
- `ENT-011` TrustProfile（解释字段、分数趋势）；`ENT-010` Feedback（反馈质量来源）；`ENT-007` Exchange（交换历史来源）；`ENT-012` Badge（徽章对信任的加成说明）。
- `INV-10` 参与方反馈权重高于社交信号（须在解释文案中显式陈述）。
- `DEC-006`（GitHub 验证作为信任骨干，说明其贡献）。
- UI：`Card`、`StatBlock`、`StatusPill`、`LineChart(accent)`、Material Symbols Outlined（`UI-003`）。

### Route or surface ID

- Surface: PAGE-040 内的抽屉/模态层，可深链 `?explain=trust`（`/u/:githubLogin?explain=trust`，便于 agent/无障碍直达，`ASM-036`）。
- 非独立全页（`Not applicable` 作为独立路由的理由：它是 PAGE-040 的解释层，须保持档案上下文）。

### Data required

- **贡献维度拆解**（来自 `ENT-011.解释`，每项：维度名、当前贡献方向/分值或占比、一句话说明）：
  - 交换历史（已完成交换数、零争议比例、`Completed`/`Closed` 占比）。
  - 反馈质量（作为参与方收到的结构化反馈均值；明确高权重）。
  - GitHub 验证（`Verified` 是否成立及其贡献，`DEC-006`）。
  - 举报/处罚（成立举报导致的扣分，中性表述，不泄露举报细节，`INV-04`）。
  - 徽章加成（`ENT-012` 对信任的标记性加成，若建模为加成项）。
- **权重声明**：显式文案「实际交换参与方的结构化反馈，对信任分的影响高于收藏/认可等社交信号」（`INV-10`）。
- **趋势归因**（可选）：折线近期变化的简要归因（如「近一月完成 3 次交换，反馈均值 4.8」）。
- **可审计指引**：链接到 `IA-013`/仓库中公开的信任评分规则说明（`NFR-004`）。

### Actions

- 关闭抽屉/模态返回 PAGE-040。
- 「了解评分规则」→ `IA-013`（仓库/帮助/关于，公开规则）。
- 各维度可展开/折叠查看其计算口径说明（纯文本，无私有数据）。

### States

- **就绪**：完整拆解。
- **解释数据缺失/计算中**：占位「信任解释生成中」（与 PAGE-040 守卫一致）。
- **新用户**：显示「尚无足够交换与反馈，信任分以基础分起步，将随交换积累」。
- **错误**：加载失败 → 重试。

### Validation and error behavior

- 拆解各分项不得出现 PII、举报方身份、私有内容（`INV-04`）；举报扣分仅以中性聚合方式呈现。
- 权重声明文案为强约束内容，不可被配置移除（`INV-10` 的界面保证）。
- 与 PAGE-040 圆环展示的总分必须一致（同一 `ENT-011` 数据源；不一致则显示「数据刷新中」而非两个矛盾数字）。

### Telemetry or analytics

- 聚合事件 `trust_breakdown_open`、`trust_rule_link_click`（无 PII，`INV-09`），并入 `FLOW-008` 口径。

### Acceptance checks

- 每个信任分至少能展开为「交换历史 / 反馈质量 / GitHub 验证 / 举报」四类来源说明（`HARD-03`）。
- 界面显式陈述「参与方反馈 > 社交信号」（`INV-10`）。
- 解释总分与 PAGE-040 圆环数字一致，无矛盾。
- 含指向公开评分规则的链接（`NFR-004`）。
- 无私有内容、无举报方身份泄露（`INV-04`）。

## PAGE-042 结构化反馈提交（子 surface — 反馈表单）

### Page purpose

让一次已完成交换的**实际参与方**提交结构化反馈（`FLOW-004`），是信任分「反馈质量」来源的录入端，也是 `INV-10` 高权重信号的产生处。本子 surface 由交换详情（`IA-006`，归 exchange 模块）在交换进入 `WaitingForFeedback` 后触发，但**反馈的维度结构、权重语义与提交校验属本（trust-feedback）模块拥有**，故在此定义其页面契约，供 exchange 模块嵌入复用。提交后驱动 `ENT-011` 重算与 PAGE-040/041 的信任更新。

> 归属说明（tradeoff）：反馈「在哪触发」属交换生命周期（exchange 模块 `IA-006`），但反馈「评什么维度、如何计权、如何校验」是信任语义的真源，归 trust-feedback。本页只定义反馈表单 surface，不重复定义交换状态机（见 BUSINESS_FLOW `FLOW-003`）。

### Upstream requirement links

- `FR-050` 结构化反馈与信任；`FLOW-004` 反馈与信任更新。
- `ENT-010` Feedback（维度：清单一致性 / 隐私边界 / 结构清晰度 / 有用性 / 再次交换意愿；公开文本；权重）。
- `ENT-007` Exchange（仅 `Completed`→`WaitingForFeedback` 的参与方可提交，`ASM-010`/`ASM-011`）。
- `INV-10`（此反馈即高权重来源）；`INV-11`（提交写 `AuditLog`）；`NFR-006`（身份核查、唯一性、速率限制）；`NFR-005`/`FLOW-007`（提交为跨边界动作，需登录身份）。
- UI：表单字段、评分控件（沿用 `UI-003`，评分用 Material Symbols Outlined star/分级，非新组件）、`PrimaryButton`、`StatusPill`、`EmptyState`。

### Route or surface ID

- Surface: 嵌入 `IA-006` 交换详情的反馈区，深链 `/exchanges/:exchangeId?feedback=1`（`ASM-036`）。
- 访问：仅该 `Exchange` 的参与方、且交换处于 `WaitingForFeedback`、且本人尚未提交过该次反馈（唯一性，`NFR-006`）。
- 非公开页（`Not applicable` 作为公开路由：反馈**提交**入口非匿名可看；反馈**结果**的公开摘要呈现在 PAGE-040 反馈质量区）。

### Data required

- 上下文（只读）：交换对方 @handle、涉及模块脱敏标题、交换状态 `StatusPill`（来自 `ENT-007`，不含私有内容）。
- 表单字段（`ENT-010` 维度，结构化评分 + 可选公开文本）：
  - 清单一致性（公开清单与私下交付是否一致，呼应 `HARD-07`）。
  - 隐私边界（对方是否尊重隐私边界）。
  - 结构清晰度。
  - 有用性。
  - 再次交换意愿。
  - 可选公开评论文本（将公开展示在对方 PAGE-040 反馈质量区，须无 PII/无私有内容）。
- 权重提示（只读说明）：告知「你的反馈作为交换参与方，对对方信任分的影响高于普通社交信号」（`INV-10`），让评价者理解其分量。

### Actions

- 逐维度评分（必填项校验）+ 填写可选公开评论。
- 提交反馈（`PrimaryButton`）→ 写 `ENT-010`、触发 `ENT-011` 重算（`FLOW-004`）、写 `AuditLog`（`INV-11`）、发通知给对方（`FLOW-006`，由通知模块承接）。
- 保存草稿（可选，`ASM-039`）/ 取消。

### States

- **可提交**：交换为 `WaitingForFeedback` 且本人未提交 → 表单可用。
- **已提交**：显示只读「已提交」态 + `StatusPill`，不可重复提交（唯一性，`NFR-006`）。
- **不可提交**：非参与方 / 交换未到反馈阶段 → 隐藏或禁用并说明原因。
- **反馈窗口已过**（`ASM-011`）：交换 `Closed` 后窗口关闭 → 只读说明「反馈窗口已结束」。
- **提交中 / 成功 toast / 错误**：loading、成功提示、校验错误内联。

### Validation and error behavior

- 必填维度校验：缺失评分不可提交，内联错误。
- 唯一性校验（`NFR-006`/类比 `INV-07`）：同一参与方对同一交换仅一份反馈；重复提交被拒。
- 资格校验：服务端二次校验提交者确为该 `Exchange` 参与方且状态合法（前端禁用不足以保证，`NFR-006`）。
- 速率限制：异常高频提交触发限制提示。
- 公开评论守卫：提交前提示评论将公开，前端做基础敏感信息（疑似邮箱/密钥/私有路径）提醒（轻量，复用隐私门理念但不替代后端校验，`INV-04`）。
- 审计：成功提交写 `AuditLog`（`INV-11`）。

### Telemetry or analytics

- 聚合事件 `feedback_submitted`（计数，含交换数聚合口径，无 PII，`INV-09`/`FLOW-008`）；不记录反馈具体评分到可识别用户的遥测明细。

### Acceptance checks

- 仅 `WaitingForFeedback` 阶段的实际参与方可提交，且每人每次交换仅一份（`NFR-006`/`ASM-010/011`）。
- 反馈维度完整覆盖 `ENT-010`（清单一致性/隐私边界/结构清晰度/有用性/再次交换意愿）。
- 提交后驱动 `ENT-011` 重算，反映到对方 PAGE-040 / PAGE-041（`FLOW-004`）。
- 提交写审计轨迹（`INV-11`）；公开评论无 PII/私有内容（`INV-04`）。
- 表单视觉锚定规范设计系统（`DEC-012`）；无经济元素（`DEC-007`）。

## PAGE-043 信任网络索引 / 着陆页

### Page purpose

公共主导航「信任网络」(`IA-007`) 的**着陆面**。此前缺口：导航项指向 `IA-007`，但本模块仅定义了 per-user 档案 `/u/:githubLogin`，点击导航无落地页（审核发现）。本页补齐：以"发现可信贡献者 + 解释信任如何运作"为目的——访客（含未登录、含 agent）在此浏览/检索社区中的可信用户、按领域主题筛选、查看精选贡献者与新晋可信成员，并理解平台的信任机制，再二级进入某人的信任档案 `PAGE-040`。**非竞争性、非付费**：不做可购买/可氪金的排他排行（`DEC-007`），任何"排名/榜单"只作可解释的发现辅助、明确口径。

### Upstream requirement links

- `IA-007`（信任网络 / 用户信任档案）— 本页是其"网络"维度的着陆面，`PAGE-040` 是其"档案"维度。
- `FR-060` 用户信任档案（聚合发现多位贡献者）；`FR-050` 结构化反馈与信任（可见信任级别/徽章作为发现维度）。
- `FR-001` 公开站点外壳与导航 / 全局搜索（用户分类结果可路由至此或档案）。
- `ENT-001` User、`ENT-011` TrustProfile（派生）、`ENT-012` Badge、`ENT-020` Topic/Tag（按领域筛选）。
- `INV-04` 公开匿名可看、不含私有内容；`INV-10` 参与方反馈权重高于社交信号（用于排序口径）；`INV-09` 任何聚合统计不含 PII。
- `DEC-006` GitHub 身份；`DEC-007` 无经济模型（无付费排名）；`DEC-012` 规范设计系统。
- UI：`UI-003` 组件 `Card`、`ListRow`/`DataTable`、`TrustBadge`、`StatBlock`、`SearchBar`、`EmptyState`、`AppShell`；`UI-001` 令牌；`UI-002` Material Symbols Outlined。

### Route or surface ID

- Route: `/trust`（呼应 shell-discovery 主导航 `信任网络 /trust(IA-007)`，消除该缺口）。
- 入口：公共主导航「信任网络」；全局搜索的"用户/贡献者"分类聚合；关于页「信任如何积累」链接。
- 出口：贡献者条目 → `PAGE-040` `/u/:githubLogin`；"信任如何形成"说明 → 复用 `PAGE-041` 解释面。
- 访问：公开匿名可看（`INV-04`）。

### Data required

只读聚合，不含 PII / 私有内容（`INV-04/09`）：
- 平台信任概览 `StatBlock` 行：可信贡献者数、已验证用户数、累计有效反馈数（聚合，`ENT-019` 口径，`INV-09`）。
- 可信贡献者列表（`ENT-001`+`ENT-011` 派生）：头像、@handle、`GitHub Verified` 徽、信任等级 `TrustBadge`、信任分（`824/1000`，`ASM-037`）、领域标签（`ENT-020`）、贡献模块数/交换数摘要。默认按"可解释信任"排序（参与方反馈权重高于社交信号，`INV-10`）。
- 筛选/检索维度：领域主题（`ENT-020`）、最低信任等级、是否 Verified、关键词（`SearchBar`）。
- 可选分区：精选贡献者、新晋可信成员、活跃领域（均为发现辅助，非排他榜）。
- "信任如何运作"说明区：链接到信任机制解释（复用 `PAGE-041` 内容/`IA-013` 关于页）。

### Actions

- 检索/筛选/排序贡献者（按领域、信任等级、Verified、关键词）。
- 点击贡献者 → 进入其信任档案 `PAGE-040`。
- 登录态：对贡献者发起"关注/认可"（与 `PAGE-040` 社交动作一致，权重低于参与方反馈 `INV-10`）。
- 查看"信任如何形成"说明（弹层/链接）。
- 无任何付费/购买排名动作（`DEC-007`）。

### States

- 默认：贡献者列表（分页/加载更多）。
- 加载：骨架屏；错误：可重试。
- 空状态（`EmptyState`）：注册表早期无足够可信用户时，说明"信任随交换与反馈积累"，引导去发现页或提交模块。
- 筛选无结果：提示放宽条件。
- 未登录 vs 登录：登录态解锁关注/认可。

### Validation and error behavior

- 排序/筛选参数非法 → 回退默认排序并提示。
- 列表只读公开数据；不得渲染任何私有内容或联系方式（`INV-04/03`）。
- 排名/榜单必须标注口径（依据哪些可解释信号），不得呈现为可付费提升（`DEC-007`/`HARD-03`）。

### Telemetry or analytics

- 仅聚合、不含 PII（`FR-140`/`FLOW-008`/`INV-09`）：着陆页访问量、筛选维度使用分布、贡献者条目点进率（聚合）。不记录个体可识别浏览行为。

### Acceptance checks

- 主导航「信任网络」点击有确定着陆页 `/trust`，不再悬空（缺口闭合）。
- 列表项可键盘可达、信任等级有文字标签（非仅颜色，`NFR-007`）。
- 列表/统计不含任何 PII 或私有内容（`INV-04/09`）。
- 任一"排名/精选"均标注可解释口径且无付费提升入口（`DEC-007`）。
- 贡献者条目可正确路由到 `/u/:githubLogin`（`PAGE-040`）。
- 视觉锚定规范设计系统（`DEC-012`：Material Symbols、IconChip、AppShell、令牌）。
